Corti publishes release notes, SBOM, and documents continuous LLM retraining
across a multi-vendor stack. When a deployer (e.g. a 911 agency) sees behavior
shift, their logs trust the declared model tag — they can't attribute drift to
vendor weights vs harness vs their own integration without a controlled A/B on
a frozen probe set.

I'm building open-source tooling (`agent-acceptance`) that produces exactly that
attribution: full agent path vs pinned vendor API on the same tag, with a signed
evidence pack mapped to EU AI Act Art. 25(2)/(4) cooperation and failure-mode
disclosure obligations (Digital Omnibus adopted, OJ pending).

For Corti specifically, this is a *vendor-rebut* instrument: when a deployer
claims your stack regressed, you can attach the same signed pack showing which
layer moved — bilateral Art. 26(5) notification, not blame-shifting.

90-day pilot on a public EMS/triage benchmark stratum plus one deployer-supplied
